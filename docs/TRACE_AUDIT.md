# Deterministic Trace Audit

## Method

The traces below are generated by the exact executable after the corrected setup sequence. Opening Active and optional Bench Pokémon are selected from the shuffled opening hand, then six Prize cards are placed from the remaining deck without another shuffle: https://www.pokemon.com/us/pokemon-tcg/rules.

Every state-changing trace line names its rule or policy IDs. Debug Prize output is never read before a legal deck or Prize inspection. The manifest records the scenario, maximum ready-by deadline, seed, and saved filename for each reviewed example.

The saved traces are independent audit examples. They are not asserted to use the same seeds or exact first-ready turns as the separately registered CTest deadline regressions in `CMakeLists.txt`.

## Reproduction

```text
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel 2
ctest --test-dir build --output-on-failure
python scripts/regenerate_setup_baselines.py --exe build/regidrago_sim --out-dir regenerated-baselines
```

CTest validates its registered deadline cases. The baseline generator separately reproduces the six saved audit traces and aggregate matrix from [`../results/baseline_manifest.json`](../results/baseline_manifest.json). The generator uses an exclusive lock and atomic file replacement.

## Boundary

This remains a setup-policy engine. It does not resolve a complete opposing board, damage, Knock Outs, Prize taking, hand disruption, gust, or all Expanded interactions.
