# Reproducible results and trace artifacts

`simulation_results.csv` is the current 100,000-trial baseline generated with seed `20260705` from the simulator source recorded by `simulator_source_sha256` in `baseline_manifest.json`. The documented reproduction command is https://github.com/FlareZ123/pokemon-sims/blob/main/README.md#run-aggregate-smoke-test.

The setup-report contract hashes every file under `src/`, including the aggregate driver in `src/trace_engine_v2/part_016.inc`. Any simulator source change invalidates the checked-in baseline until the matrix, manifest, and generated reports are refreshed: https://github.com/FlareZ123/pokemon-sims/issues/642.

`traces/` contains deterministic `--simulate-this` transcripts used for manual review. Each state-changing line includes the rule IDs documented in [`../docs/RULES_TRACEABILITY.md`](../docs/RULES_TRACEABILITY.md).

The previous `variant_results.csv` screen was removed because it was produced before the trace-policy rewrite. It must not be reused until variants are regenerated by this engine.

The C++ writer uses a destination lock and atomic replacement. The Python generators use exclusive locks and atomic replacement. CTest covers the trace matrix, core fixtures, Tier Two policy fixtures, and the aggregate smoke run.
